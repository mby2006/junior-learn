## 一、UI 设计层优化

### 1.1 色彩系统

**现状问题**

| 问题 | 描述 |
|------|------|
| 主色过多 | 同时使用蓝 `#4F8CFF`、粉 `#F472B6`、绿 `#10B981` 等多色，视觉重心分散 |
| 渐变过重 | `btn-neon`、`mastery-fill`、`grad` 文字、header 背景均使用线性渐变，页面"闪"感强 |
| 文字颜色语义混乱 | `text-slate-800` 被混用于次要文字、提示、标签，缺乏层次区分 |
| 科目颜色重复 | `b-physics` 和 `b-politics` 均使用粉色系，区分度低 |

**优化方案**

```
主色（Primary）：  #4F8CFF  —— 所有主操作、链接、激活态
辅色（Accent）：   #F472B6  —— 仅用于特殊强调（streak火焰、特殊角标）
成功（Success）：  #10B981  —— 正确、掌握、在线状态
警告（Warning）：  #F59E0B  —— 偏弱、注意
危险（Danger）：   #EF4444  —— 错误、弱项
文字层次：
  主文字：  #1E293B（现有）
  次文字：  #64748B（现用作次要）→ 统一为 #475569
  提示文字：#94A3B8（新增，替代部分 slate-800 误用）
```

- 所有 `gradient` 渐变按钮改为纯色 `#4F8CFF` 实心按钮，hover 时降低不透明度至 `.88`
- `grad` 渐变文字仅保留 Logo，标题一律使用 `#1E293B`
- 政治科目颜色改为紫色 `#8B5CF6`，消除与物理的混淆

### 1.2 排版与字体

**现状问题**

- `font-size` 层次混乱：`.7rem`、`.63rem`、`.55rem` 等超小字号在信息密度高处频繁出现，移动端阅读困难
- `font-weight:700` 被大量使用（`font-bold`），加粗层次失效
- 行距不一致：`.md p` 设 `line-height:1.8`，普通正文无设置，默认 `1.5`，排版割裂
- `font-mono` 被用于非数字场景（"系统状态"标签等），显得突兀

**优化方案**

```
字号规范（4级）：
  标题/页面名称：  15px / font-weight:600
  卡片标题：       14px / font-weight:500
  正文内容：       13px / font-weight:400
  辅助/角标：      12px / font-weight:400（最小不低于 12px）

  ❌ 禁止使用 .7rem（11.2px）、.63rem（10px）、.55rem（8.8px）

行距：
  标题：line-height: 1.4
  正文：line-height: 1.7
  代码：line-height: 1.6

font-mono 仅用于：数字（分数、百分比、时间）、代码块
```

### 1.3 布局结构

**现状问题**

- 侧边栏宽度 `w-56 (224px)` + 主区内容区 `p-4 md:p-8`，在 1280px 以下屏幕有挤压感
- Header 高度在不同页面不一致（有些页面 `py-3`，有些 `py-4`）
- Dashboard 卡片 `grid-cols-3` 在 1024–1280px 区间三栏过窄，中间栏内容截断
- 页面切换无任何过渡动画，视觉跳跃感强

**优化方案**

```
侧边栏：
  - 宽度调整为 w-52 (208px)，释放主内容区空间
  - 侧边栏底部 API 地址输入框移至「设置」页，侧边栏底部仅保留状态统计

Header：
  - 高度固定为 56px，所有页面统一
  - 背景去除渐变，改为 bg-white + border-b

页面切换：
  - 添加 x-transition:enter / x-transition:leave
  - 进入：opacity 0→1 + translateY 4px→0，200ms ease-out
  - 离开：opacity 1→0，100ms

卡片网格：
  - Dashboard Block 2 改为 md:grid-cols-[1.5fr_1fr]，给科目正确率更多宽度
  - 所有卡片 padding 统一为 p-5（20px），去除 p-4/p-6 的混用
```

### 1.4 组件设计

#### 导航按钮（nav-btn）

**现状**：激活态使用 `border-left: 2px solid #4F8CFF`，风格偏老；hover 时粉色描边与激活态蓝色不统一。

**优化**：
```css
/* 激活态：填充背景 + 左侧粗线 */
.nav-btn.active {
  background: rgba(79,140,255,.10);
  border-left: 3px solid #4F8CFF;
  color: #4F8CFF;
}
/* hover 统一为浅蓝，去掉粉色 */
.nav-btn:hover:not(.active) {
  background: rgba(79,140,255,.05);
  color: #334155;
}
```

#### 按钮体系

**现状**：`btn-neon`（渐变+光晕）、`btn-ghost`、`pill` 三套按钮风格混搭无规律。

**优化**：整理为 3 个语义级别：

```
btn-primary:  背景 #4F8CFF，白色文字，圆角 8px，主操作
btn-outline:  边框 1px solid #4F8CFF，蓝色文字，圆角 8px，次要操作  
btn-ghost:    无边框，灰色文字，hover 时浅灰背景，第三级操作
```

去除所有 `box-shadow: 0 0 20px rgba(...)` 光晕效果。

#### 卡片（card）

**现状**：hover 有 `translateY(-1px) + box-shadow 放大`，所有卡片均有此效果，包括纯展示卡片。

**优化**：
- 可点击卡片：保留 hover 上浮效果
- 纯展示卡片（如 dashboard 分析卡）：移除 hover 效果，避免误操作感
- 添加 `cursor: default` 区分

#### Badge / 科目标签

**现状**：`padding: 2px 8px; font-size: .7rem`，在 retina 屏可读，但普通屏偏小。

**优化**：`padding: 3px 8px; font-size: 12px`，最小可读尺寸保障。

#### 输入框与下拉框

**现状**：各处输入框样式不统一，有些 `border:1px solid #E8EDF4`，有些用 Tailwind `ring`，有些 inline style。

**优化**：
```css
/* 统一输入框规范 */
.input-base {
  height: 36px;
  padding: 0 12px;
  border: 1px solid #E2E8F0;
  border-radius: 8px;
  font-size: 14px;
  outline: none;
  transition: border-color .15s;
}
.input-base:focus { border-color: #4F8CFF; }
```

### 1.5 动效与交互反馈

**现状问题**

- 动画不一致：有的用 `fadein`（`translateY 6px`），有的 `flipIn`（`translateY 14px`），有的直接 `x-show` 无动画
- `spin` 动画时长 `.75s` 偏快，AI 处理时视觉焦虑感增强
- 没有操作成功/失败的 Toast 提示，用户只能看 alert

**优化方案**

```
统一动画时长：
  页面进入：200ms ease-out（translateY 4px → 0）
  卡片展开：250ms ease
  模态框：180ms ease-in-out（scale .96 → 1）
  加载 spinner：1s linear（降速减轻焦虑感）

新增 Toast 组件（替代 alert）：
  - 右上角固定定位
  - success（绿）/ error（红）/ info（蓝）三种类型
  - 自动 3s 后消失，支持手动关闭
  - Alpine.js 全局 $store.toast.show(msg, type) 调用
```

### 1.6 移动端适配

**现状问题**

- 底部 Tab Bar 的文字 `font-size: .55rem (8.8px)` 完全不可读
- 移动端 `sess-sidebar` / `hw-sidebar` 抽屉打开时没有背景遮罩动画（仅 box-shadow）
- 部分页面 `mob-scroll` padding-bottom 过大（`5.5rem`），底部留白浪费
- 上传区域在移动端缺少"点击拍照"的明确引导

**优化方案**

```
底部 Tab Bar：
  - 图标改用 SVG（替代 emoji，避免系统字体差异）
  - 文字字号：font-size: 11px（最小可读）
  - 激活态：图标+文字均着色，加一个上滑指示条（现有）
  - Tab 数量限制为 5 个（dashboard、homework、inquire、wrongbook、more）

移动端抽屉：
  - 添加半透明遮罩层（opacity: .4 的黑色背景）
  - 遮罩点击关闭抽屉

上传区域移动端引导：
  - "拍照" / "从相册选择" 两个按钮分开展示
  - accept="image/*" capture="environment" 触发摄像头
```

## 二、功能层优化

### 2.1 作业批改页

**现有流程**：上传图片 → 选科目 → 批改 → 显示结果 + 知识点 → 可加入错题本

**问题与优化**

| 问题 | 优化方案 |
|------|----------|
| 每次批改只能上传 1 张图片 | 支持多图上传（同一份作业多页），批量合并批改 |
| 批改结果只显示文字，无结构化分题展示 | 结果改为「题目卡片」列表：每题单独显示答案、对错、解析 |
| 重新批改会覆盖历史，无法对比 | 侧边栏历史列表支持「对比两次批改」的 diff 视图 |
| 加入错题本需手动点按钮 | 提供「自动加入错题」开关（默认开启），批改完直接入库 |
| 没有评分展示 | 批改结果顶部增加分数卡：得分 / 满分 / 百分比，大号显示 |
| 知识点芯片无分组 | 知识点按科目 → 章节 两级分组展示，折叠/展开 |

---

### 2.2 知识点答疑页

**现有流程**：输入问题（可附图）→ AI 流式回答 → 可追问最多 5 轮

**问题与优化**

| 问题 | 优化方案 |
|------|----------|
| 5 轮限制不够，打断思路 | 改为 10 轮，或允许用户手动「延伸讨论」 |
| 会话列表只显示第一句话，无科目标签 | 会话列表增加科目徽章 + 时间 + 题目预览（截取前 30 字）|
| 没有「收藏此解答」功能 | 添加收藏按钮，收藏内容存入专属「笔记本」视图 |
| 输入框没有快捷模板 | 添加常用提问模板按钮：「帮我解题」「讲解概念」「举个例子」 |
| 图片上传后无预览缩略图 | 输入区下方显示已附图缩略图，点击可删除 |
| 搜索功能缺失 | 会话列表顶部增加搜索框，支持按内容全文搜索 |
| RAG 来源不透明 | 回答底部显示「参考教材」折叠区（来自哪本书哪一章）|

---

### 2.3 智能出题页

**现有功能**：自定义模式（知识库出题）、模拟试卷模式（PDF 仿写）、WebSocket 流式进度

**问题与优化**

| 问题 | 优化方案 |
|------|----------|
| 出题完成后题目只在 data/ 文件夹，前端无展示 | 出题完成后直接渲染题目卡片，支持在线作答 |
| 在线作答缺失 | 每道题增加「作答区」，提交后对比正确答案并解析 |
| 出题参数（难度、类型）设置区 UI 太紧凑 | 重新排列为清晰的表单布局，加入难度滑块（简单/中等/困难）|
| 没有出题历史 | 保存每次出题记录，可重新查看和重做历史题目 |
| 模拟试卷模式需上传 PDF，无其他方式 | 增加「从历史批改记录」生成仿真卷的快捷入口 |

### 2.4 错题本页

**现有功能**：CRUD、按科目/状态筛选、知识点标签

**问题与优化**

| 问题 | 优化方案 |
|------|----------|
| 列表视图缺少「掌握程度」进度条 | 每道错题卡片增加掌握率进度条（0%~100%，可手动更新）|
| 没有「复习模式」 | 新增「开始复习」按钮，进入单题翻卡模式（正面题目→背面解析）|
| 没有遗忘曲线 / 间隔复习提醒 | 基于艾宾浩斯遗忘曲线，标记「今日需复习」的题目数量 |
| 知识点筛选只能单选 | 支持多选知识点组合筛选 |
| 批量操作（批量标记掌握/删除）缺失 | 添加 checkbox 多选 + 批量操作工具栏 |
| 搜索功能缺失 | 顶部增加搜索框，支持搜索题目文字内容 |

### 2.5 Dashboard 主页

**现有功能**：学情总结、科目正确率柱状图、薄弱知识点、学习记录时间线、系统功能导引

**问题与优化**

| 问题 | 优化方案 |
|------|----------|
| 没有今日任务模块 | 添加「今日待办」卡片：今日推荐复习题数 + 快捷完成按钮 |
| 学习连续天数（streak）只展示数字 | 添加 7 日热力格（每格代表一天，有无学习），直观视觉化 |
| 「使用指南」流程卡占屏幕空间较大 | 改为可折叠模块，首次访问展开，后续默认折叠 |
| 科目正确率柱状图数据为空时占位太大 | 空状态改为更简洁的引导文字 + 单个操作按钮 |
| 没有本周学习统计 | 添加「本周」小结：批改次数、答疑次数、新增错题数 |

### 2.6 学情档案页

| 学习目标字段缺失 | 新增「本学期目标」文本框 + 进度追踪 |

### 2.7 新增功能建议

以下是目前系统没有、但对初中生学习很有价值的功能，按实现难度排序：

#### 难度 ⭐（1~2 天可实现）

**A. 暗色主题**  
在侧边栏底部添加亮/暗切换按钮，切换 CSS 变量集。对眼睛保护重要，也是青少年用户喜爱的功能。

**B. 全局快捷键**  
- `Alt+H` → 跳转作业批改
- `Alt+Q` → 跳转答疑
- 输入框内 `Ctrl+Enter` → 提交

**C. 今日复习通知**  
页面 title 显示「(3 题待复习) StudyCore」，提醒用户未完成的错题复习。

#### 难度 ⭐⭐（3~5 天可实现）

**D. 单词本**  
针对英语科目的专项模块：生词收集 → 遗忘曲线复习 → 例句查看。英语词汇薄弱是初中生常见痛点。

**E. 错题卡翻卡复习模式**  
进入全屏翻卡模式：正面显示题目，点击翻转显示解析，左划「不会」右划「会了」，记录掌握情况。

**F. 学习计划功能**  
设置每周学习目标（如：每天复习 5 道错题 + 提问 2 次），仪表盘展示完成进度环。

#### 难度 ⭐⭐⭐（1~2 周可实现）

**G. 公式手写识别**  
利用现有 LLM Vision 能力，在答疑框加入「手写板」：用鼠标/触屏手写数学公式，识别成 LaTeX 后提问。

**H. 同步学习笔记**  
在答疑页添加「边学边记」侧边面板，答疑过程中随时摘录重点，自动关联科目和知识点，生成结构化笔记。

**I. 错题卡导出 PDF**  
选择错题后，一键导出为 A4 格式 PDF（包含题目、标准答案、解析），可打印复习。

## 三、工程与性能

| 问题 | 优化方案 |
|------|----------|
| `index.html` 已达 3867 行，难以维护 | 将各页面模块拆分到独立 `.html` 片段，通过 `<template>` 或 ES Module 动态注入 |
| 所有 JS 逻辑在一个 `app()` 函数中，约 2000+ 行 | 按功能拆分为 `homeworkModule()`、`inquireModule()` 等，Alpine.js `$data` 混合注入 |
| `vendor/` 文件无版本锁定 | 在 vendor 文件名加版本号（e.g. `alpine.3.14.8.min.js`），避免意外更新 |
| 图片上传无压缩 | 上传前在浏览器端用 `canvas` 压缩到 max 1600px，减少服务器 OCR 处理时间 |
| WebSocket 连接异常无重连机制 | 添加指数退避重连（1s → 2s → 4s → 最多 30s），断线后显示重连状态 |
| 无任何前端错误上报 | 添加简单的 `window.onerror` 捕获，显示友好错误页而非白屏 |
